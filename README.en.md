# 📚 llmWikiX: Minimal LLM Knowledge Base System

llmWikiX is a minimal local knowledge base that exposes Markdown knowledge files to external AI Agents via MCP (Model Context Protocol) server for search and retrieval.

## 🌟 Core Philosophy

- **Zero Configuration**: No database, no vector engine, no environment variables — pure filesystem-driven
- **MCP Standard Protocol**: Expose your knowledge base via MCP, and any MCP-compatible AI Agent can connect instantly
- **Markdown is Everything**: Manage knowledge in the most universal format — just drop `.md` files into `content/knowledge-base/`

## 🏗 Architecture

```
content/knowledge-base/*.md   ← User-maintained Markdown knowledge files
        │
        ▼
content/wiki/                 ← Auto-generated by wiki-creater Skill
        │
        ▼
┌─────────────────────────┐
│  kb-mcp-server.mjs      │  MCP stdio server (plain-text index + keyword search)
│  kb-mcp-http-server.mjs │  HTTP bridge (:8787)
└─────────┬───────────────┘
          │
          ▼
   External AI Agent / IDE / LLM Tool
   (GitHub Copilot, Claude Code, Codex, ...)
```

## ✨ Features

- 🔍 **Keyword Search**: Weighted scoring with title(×8) + description(×4) + content(×2) for precise matching
- 📖 **Entry Reading**: Read full entry content via `kb://` URI or relative path
- 📋 **Entry Listing**: List all available entries by scope (knowledge-base / wiki)
- 🔄 **Index Refresh**: Automatic mtime-based incremental detection + 5-minute TTL cache, or manual force refresh
- 🌐 **Dual Transport**: Native MCP stdio + HTTP JSON-RPC bridge for different integration scenarios

## 📦 MCP Tools

| Tool | Description |
|------|-------------|
| `search_knowledge_base` | Search the knowledge base by keyword with scope filtering |
| `read_knowledge_base_entry` | Read an entry by URI (`kb://scope/path`) or relative path |
| `list_knowledge_base_entries` | List all available entries |
| `refresh_index` | Force refresh the index cache |

## 🚀 Quick Start

### 1. Install

```bash
git clone https://github.com/TreeX-X/llmWikiX.git
cd llmWikiX
npm install
```

### 2. Add Knowledge Files

Drop your Markdown files into `content/knowledge-base/`. Subdirectories are supported for categorization:

```
content/knowledge-base/
├── ai-tips/
│   └── prompt-engineering.md
├── dev-experience/
│   └── vscode-tips.md
└── my-notes.md
```

Markdown file frontmatter example:

```yaml
---
title: "Prompt Engineering Tips"
date: 2026-05-07
description: "Common prompt design patterns and best practices"
tags: ["AI", "prompt"]
---

# Prompt Engineering Tips

Body content...
```

### 3. Generate Wiki Index (Optional)

Use the [wiki-creater Skill](https://github.com/TreeX-X/llmWikiX) to auto-generate concept and source indexes under `content/wiki/`.

### 4. Start MCP Server

**stdio mode** (recommended for IDE integration):

```bash
npm run mcp
```

**HTTP mode** (for network access):

```bash
npm run mcp:http
# Listening at http://127.0.0.1:8787/mcp
```

## 🔧 IDE Integration

### GitHub Copilot / VS Code

Add to `.vscode/mcp.json`:

```json
{
  "servers": {
    "llmwiki": {
      "command": "node",
      "args": ["scripts/kb-mcp-server.mjs"],
      "cwd": "${workspaceFolder}"
    }
  }
}
```

### Claude Desktop

Add to `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "llmwiki": {
      "command": "node",
      "args": ["/path/to/llmWikiX/scripts/kb-mcp-server.mjs"]
    }
  }
}
```

### HTTP Access

```bash
# Health check
curl http://127.0.0.1:8787/health

# Search
curl -X POST http://127.0.0.1:8787/mcp \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"search_knowledge_base","arguments":{"query":"MCP","limit":5}}}'
```

## 📐 URI Convention

Knowledge base entries are identified by `kb://` URIs:

```
kb://knowledge-base/ai-tips/prompt-engineering.md
kb://wiki/concepts/roller-cutter.md
```

- `kb://` — fixed protocol prefix
- `knowledge-base` or `wiki` — scope identifier
- Path portion is URL-encoded

## 🤝 Integration with WorkflowX

llmWikiX integrates seamlessly with the [WorkflowX](https://github.com/TreeX-X/workflowX) multi-agent workflow:

- **PlannerX** output PRDs can be placed directly into `content/knowledge-base/`
- **CoderX** knowledge artifacts from development are exposed via MCP to downstream agents
- **EvaluatorX** can search the knowledge base through MCP for contextual assessment
